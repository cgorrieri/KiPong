Comment tester votre projet DeViNT et quelles sont les consignes que vous devez respecter

Jean-Paul Stromboni pour l’équipe des projets DeViNT, le 9 mai 2011

 

Nous vous serions reconnaissants de lire ces consignes attentivement et de les respecter ce qui nous fera gagner beaucoup de temps lors de la constitution du CD DeViNT 2012.

Test préliminaire au rendu de vos projets DeViNT :

C’est un test à faire avant la séance de rendu de la semaine prochaine pour nous faire gagner du temps :

 

1.       télécharger depuis le dossier local AtelierDeViNT les fichiers au format zip de Modele_de_jeu.zip, Listor.zip (et non pas DListor.zip),VocalyzeSIVOX.zip et jre-lib_win.zip (Attention, ces fichiers ont été mis à jour récemment !)

2.       décompresser ces fichiers dans le répertoire de travail au niveau du répertoire de votre projet DeViNT, vous avez donc maintenant dans ce répertoire : votre projet, VocalyzeSIVOX, Listor, jre, et lib.

3.       exécuter Listor/bin/execution.bat ouvre un menu qui doit permettre de lancer Modele_de_jeu, mais aussi VocalyzeSIVOX, et même de lancer votre projet dans la mesure où il respecte les normes suivantes.

Quelles sont les normes que votre projet doit respecter absolument ?

La plupart de ces normes et contraintes sont matérialisées dans Modele_de_jeu qui constitue un modèle et un point de départ éventuellement:

 

1.       votre projet doit contenir quatre répertoires nommés bin, doc, src et ressources

2.       bin contient les exécutables et un fichier execution.bat capable d'exécuter le projet

3.       Nouveau ! la première ligne du fichier execution.bat est dorénavant : set PATH=../../lib/;../../jre/bin/

4.       Nouveau ! la ligne qui exécute le projet mentionne au besoin les librairies dll utilisées, lesquelles doivent être placées dans le dossier ressources/lib. Par exemple, pour Modele_de_jeu, c'est
java -cp .;../../VocalyzeSIVOX/bin/SI_VOX.jar -Djava.library.path=../ressources/lib jeu.LancementJeu

5.       doc contient les informations sur le projet, dans les fichiers 'entete.txt' pour la licence et 'infos.txt' pour l'aide; ale plus simple est d’adapter les fichiers entete.txt et infos.xml de Modele_de_jeu à votre projet. Si vous avez un document pdf ou autre donnat des informations supplémentaires, placez le dans doc.

6.       ressources contient les paramètres de la synthèse vocale, les sons et les images utilisés,

a.       dans donneesMbrola, à recopier de Modele_de_jeu,

b.      dans lib, pour les librairies dll, 

c.       le fichier texte si_vox_ihm.conf,

d.      des dossiers pour les sons et pour les images utilisés par le projet, qui doivent être libres de droits, à préciser dans un fichier droits.html placé dans ressources

7.       src contient les sources, et un fichier compilation.bat capable de créer l’exécutable du projet dans bin.

8.       Nouveau ! pour les fichiers (scores, paramètres) écrits par le projet, il faut choisir une zone du disque où le projet a les droits d'écriture. Pour savoir comment procéder avec System.getProperty("user.dir"), exécuter le menu Fichier de Modele_de_Jeu. La nouvelle version de VocalyzeSIVOX le fait maintenant, pour sauvegarder le fichier Wave qu'elle va jouer ensuite.

9.       un petit truc utile, si un mot est mal prononcé par SIVOX, vous pouvez entre autres aller modifier le fichier ‘exceptions.txt’ dans ressources/donneesMbrola/rules. Par exemple pour lire le mot ‘fichier’, on y a ajouté la règle suivante : T [[ fichier ]] T -> f i S i e          ## fichier (commentaire)

11. et surtout, ne négligez pas l'aide vocale de vos projets, F1 annonce la nature et le but du jeu, F2, F3, comment jouer, les règles, ... la synthèse vocale doit être réactive aux actions de l’utilisateur, etc, etc … comme déjà expliqué, et à retrouver dans la page d’information des projets (consignes.html)
